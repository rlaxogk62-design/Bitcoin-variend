🕵️‍♂️ Bitcoin Pattern Mining: From Black-box to Rule-based Strategy📝 Background (왜 이 프로젝트를 시작했는가)기존의 딥러닝 기반 가격 예측 모델(CNN-LSTM)을 통해 비트코인 종가를 예측하려 시도함. 그러나 '내일의 가격'을 정확히 맞추는 것은 블랙박스(Black box)적 성격이 강해, 진입 근거가 불명확하다는 한계를 느낌.단순히 "AI가 사라고 했으니까 산다"가 아니라, "이런 특정 패턴이 발견되었으니 수익 확률이 높아서 진입한다"는 명확한 전략이 필요하다고 판단하여, AI를 활용한 '규칙 발견(Pattern Mining) 모델'로 방향을 전환함.🛠 Tech Journey & Troubleshooting (개발 과정 및 트러블슈팅)데이터 과학은 모델 설계보다 '데이터 정제' 과정이 90%라는 것을 몸소 체험함.1. 딥러닝 vs 결정 트리(Decision Tree)목표: 복잡한 수치 예측이 아니라, 인간이 직관적으로 이해할 수 있는 'if-then' 형태의 규칙을 자동 추출하는 것.선택: DecisionTreeClassifier를 도입하여, 어떤 조건에서 승률(Class 1: 0.3% 상승)이 높은지 기계가 스스로 판단하게 함.2. 마주했던 에러와 해결 과정 (The Debugging Diary)개발 과정에서 3번의 큰 벽을 만났고, 모두 스스로 해결함.Error 1 (Missing Library): ModuleNotFoundError: No module named 'pyupbit'원인: 코랩 환경에 라이브러리 미설치.해결: !pip install pyupbit로 설치 자동화 파이프라인 구축.Error 2 (Data Type): TypeError: float() argument must be a string or a real number, not 'ellipsis'원인: 데이터 정제 과정에서 발생한 inf(무한대)값과 NaN을 방치한 채 모델에 주입함.해결: df.replace([np.inf, -np.inf], np.nan) 후 dropna()를 통해 '순수 숫자 데이터'만 남도록 전처리 로직 강화.Error 3 (Sample Size): ValueError: Found array with 0 sample(s)원인: 과도한 dropna()로 인해 모델이 학습할 데이터가 0개가 되어버림.해결: 데이터 정제 후 len(df)를 체크하는 검증 로직을 추가하여, 데이터 부족 시 경고를 띄우도록 방어 코드 설계.💡 AI가 찾아낸 핵심 패턴 (Insights)학습 데이터 1,987개를 분석한 결과, AI는 다음과 같은 3가지 고승률 패턴을 제안함.규칙 조건결과 (Class 1)해석volume_diff <= 0.14 & 36.8 < rsi < 47.3매수거래량이 적고 과매도가 아닌 안정적 구간에서 기술적 반등 발생volume_diff > 0.14 & rsi <= 2.50매수극단적 과매도 상태(패닉셀)에서의 강한 반등 구간volume_diff > 7.03 & rsi > 2.50매수평소보다 7배 이상 거래량이 터지는 슈팅 구간🚀 Future Plan (향후 계획)현재 AI가 찾아낸 규칙을 기반으로, 실제 매매 차트 위에 'BUY' 신호가 자동으로 찍히도록 시각화 모듈 연동.찾아낸 규칙들의 백테스팅(Back-testing)을 통해, 실제 역사적 데이터에서 얼마나 높은 수익률을 기록했는지 검증하는 파이프라인 구축 예정.
